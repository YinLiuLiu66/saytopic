# 🎙️ SayTopic — 开发文档

## 1. 项目概述

一个在线录音 + 图片上传 + 二维码声纹打印工具：
- 用户在浏览器中直接录音 → 录音完成后自动上传到后端存储，返回可公开访问的 URL
- 录音完成后首页展示一个二维码（指向手机端图片上传页），用户扫码后在手机端上传图片；也可跳过直接生成明信片
- 手机上传图片后，首页通过轮询实时同步显示图片
- 前端生成音频二维码（指向播放页面）+ 波形图
- 点击打印按钮 → 新窗口预览，148mm×100mm 明信片尺寸打印（含图片、音频二维码和波形图）

## 4. 后端 API 设计

### 4.1 `POST /api/upload`

| 项目 | 内容 |
|------|------|
| Content-Type | `multipart/form-data` |
| 参数 | `file` — 录音文件（mp4/webm/ogg） |
| 成功响应 | `{ "url": "/api/audio/xxx.m4a", "filename": "xxx.m4a" }` |
| 错误响应 | `{ "error": "..." }` |
| 逻辑 | 保存到 `uploads/`，用 UUID 重命名，保留原始扩展名 |

### 4.2 `GET /api/audio/{filename}`

| 项目 | 内容 |
|------|------|
| 响应 | 音频二进制流（根据扩展名自动设置 Content-Type） |
| 逻辑 | 读取 `uploads/{filename}` 返回文件流 |

### 4.3 `POST /api/upload-image`

| 项目 | 内容 |
|------|------|
| Content-Type | `multipart/form-data` |
| 参数 | `file` — 图片文件（jpg/jpeg/png/webp）；`audio_filename` — 可选，关联的音频文件名（提供时图片以音频基名+图片扩展名命名） |
| 成功响应 | `{ "url": "/api/image/xxx.{ext}", "filename": "xxx.{ext}" }` |
| 错误响应 | `{ "error": "..." }` |
| 逻辑 | 保存到 `uploads/`，若提供 `audio_filename` 则以音频基名命名图片，否则用 UUID |
| 限制 | 最大 10MB，支持格式：jpg/jpeg/png/webp |

### 4.4 `GET / HEAD /api/image/{filename}`

| 项目 | 内容 |
|------|------|
| 方法 | GET, HEAD |
| 响应 | 图片二进制流（根据扩展名自动设置 Content-Type），HEAD 返回不含 body 的响应头（用于轮询检测） |
| 逻辑 | 读取 `uploads/{filename}` 返回文件流 |

### 4.5 CORS

允许前端开发服务器（`http://localhost:5173`）和公网域名（内网穿透时需添加穿透域名）跨域访问。

## 5. 前端路由

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | `HomeView` | 录音页：点击录音 → 展示图片上传二维码（可跳过）→ 轮询同步图片 → 生成明信片（音频二维码 + 波形图 + 可选图片）→ 打印按钮 |
| `/play/:filename` | `PlayView` | 播放页：QR 码扫码后跳转，`<audio>` 标签播放音频 |
| `/upload-image/:audioFilename` | `ImageUploadView` | 图片上传页：手机扫码后跳转，支持拍照和相册上传 |

## 6. 前端组件详情

### 6.1 `AudioRecorder.vue`

- 复古电话亭造型：天线、圆顶（SAY TOPIC 标签）、金属质感主体、底座
- LED 状态指示灯（等待/录音中/完成三种状态）
- 使用浏览器 MediaRecorder API 进行录音
- 优先选择 mp4 格式（iOS 兼容），其次 webm/ogg
- 录音时长 30-60 秒，带进度条，到 60 秒自动停止
- 脉冲环动画 + 麦克风图标（录音中）/ 浮动动画（等待）
- 录音完成后显示回放音频控件
- 上传按钮文案为"生成明信片"，emit `url` 和 `filename` 给父组件

### 6.2 `ImageUploader.vue`

- 供 `ImageUploadView.vue`（手机端图片上传页）使用
- 提供图片选择（`<input type="file" accept="image/*">`）和拖拽上传区域
- 支持 JPG/JPEG、PNG、WebP 格式，最大 10MB
- 图片预览功能（使用 `URL.createObjectURL`）
- 上传到 `POST /api/upload-image`
- 上传中显示进度状态
- 成功后 emit `url` 和 `filename` 给父组件
- ⚠️ 此组件**不再**出现在首页 HomeView 中，仅用于手机端扫码上传页

### 6.3 `WaveformCanvas.vue`

- Props: `audioUrl`（必填）、`height`（默认 200）、`width`（默认 600）、`colorTheme`（primary/coral）
- 使用 `AudioContext.decodeAudioData()` 解码音频
- 在 Canvas 上绘制波形（时间-振幅），带填充渐变和中线虚线
- 响应式，尺寸适配明信片打印比例

### 6.4 `QrCodeCard.vue`（打印核心）

- 使用 `mingxinpian.png` 作为底图，绝对定位叠加元素
- 尺寸：148mm × 100mm（标准明信片尺寸）
- 三个绝对定位元素：
  - 图片区：对应底图蓝色虚线框区域（`left: 5.5%`, `top: 12%`）
  - 波形区：位于图片下方（`left: 11%`, `top: 67%`）
  - 二维码：位于右侧留白区（`right: 22.5%`, `top: 33%`）
- 打印按钮在组件外部，打印时隐藏
- Props: `audioUrl`, `filename`, `imageUrl`（可选）
- 生成**一个**二维码（音频播放链接）
- ⚠️ 明信片上**不包含**图片上传二维码

### 6.5 `ImageUploadView.vue`

- 从路由参数获取 `audioFilename`
- 移动端优化界面，支持拍照和相册选择
- 使用 `<input type="file" accept="image/*" capture="environment">`
- 内部使用 `ImageUploader.vue` 组件
- 上传成功后显示成功提示
- 简洁页面，适合移动端扫码浏览
- 上传时携带 `audioFilename` 参数，后端以音频基名+图片扩展名命名，实现文件名关联

### 6.6 `PlayView.vue`

- 从路由参数获取 `filename`
- 构建音频 URL：`/api/audio/{filename}`
- 使用浏览器原生 `<audio controls>` 播放
- 简洁页面，适合移动端扫码浏览

### 6.7 `HomeView.vue` — 首页完整流程

- 录音完成后（`AudioRecorder` 组件 `@uploaded` 事件触发）：
  1. 展示双栏上传选项：
     - **电脑拍照**：使用 `CameraCapture` 组件，通过 `getUserMedia` 打开摄像头，实时预览 + 点击拍照 + 自动上传
     - **扫码上传**：展示二维码，指向 `/upload-image/{filename}`（手机扫码上传图片）
     - 两栏之间显示「或」分隔符
  2. 同时保留"跳过，直接生成明信片"按钮
- 轮询同步机制：
  - 开始每 3 秒轮询 `HEAD /api/image/{base}.{ext}`，依次尝试所有允许的图片扩展名（`.jpg`, `.jpeg`, `.png`, `.webp`）
  - 轮询到图片存在（HTTP 200）后停止轮询，将图片 URL 传入 `QrCodeCard`
  - 如果用户点击跳过，停止轮询并直接显示 `QrCodeCard`（不带图片）
- 应用户选择"跳过"或图片轮询成功后，显示 `QrCodeCard.vue`：
  - 传入 `audioUrl`, `filename`, `imageUrl`（如有）
  - 明信片包含：音频二维码 + 波形图 + 可选图片

### 6.8 `CameraCapture.vue`

- 供 `HomeView.vue` 使用，提供电脑端摄像头拍照功能
- Props: `audioFilename`（必填）
- Emits: `captured({ url, filename })` — 拍照并上传成功后触发
- 使用 `navigator.mediaDevices.getUserMedia()` 打开摄像头，实时预览（镜像翻转）
- 流程：点击拍照按钮 → canvas 捕获当前视频帧 → 转为 JPEG（quality 0.9）→ 自动上传到 `POST /api/upload-image`（携带 `audio_filename`）
- 三种 UI 状态：加载中（spinner）、不可用（权限拒绝/无摄像头错误提示）、就绪（视频预览 + 拍照按钮）
- 上传中按钮禁用，防止重复点击
- 修复要点：`cameraLoading = false` 需在 `srcObject` 赋值前执行，配合 `nextTick()` 确保 `<video>` 元素已渲染到 DOM

## 8. 打印策略

明信片通过**新窗口独立页面**实现打印，彻底规避父容器样式干扰和浏览器页眉页脚问题：

1. 用户点击"打印明信片"按钮 → `QrCodeCard.print()`
2. 获取波形图 Canvas 的 `toDataURL()` 和已有的二维码 data URL
3. 打开新窗口（约 650×500），写入完全独立的 HTML 文档（内联所有 CSS）
4. 新窗口中展示明信片预览 + "打印明信片"按钮
5. 用户点击按钮触发 `window.print()`，`@page { size: 148mm 100mm; margin: 0; }`
6. 打印完成后窗口保持打开，用户可自行关闭

```css
/* 新窗口内联样式中的打印配置 */
@media print {
  body { background: white; padding: 0; }
  .no-print { display: none !important; }
  .postcard { box-shadow: none !important; border: none !important; }
}
@page { size: 148mm 100mm; margin: 0; }
```
